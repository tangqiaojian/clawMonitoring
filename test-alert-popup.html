🔔 告警弹窗功能测试工具

📋 测试前准备

请确保前端开发服务器正在运行：http://localhost:5173/

🧪 测试方法一：手动调整阈值触发告警

步骤 1：打开监控页面

访问 http://localhost:5173/

步骤 2：调低阈值触发告警

在"阈值设置"区域将各项阈值调低到当前系统指标以下：

  • CPU 阈值调到 10% （确保低于当前 CPU 使用率）
  • 内存阈值调到 30% （确保低于当前内存使用率）
  • GPU 阈值调到 10% （如果有 GPU 的话）

步骤 3：观察弹窗

右上角应该会自动滑出告警弹窗，显示对应的告警信息

步骤 4：测试关闭功能

点击弹窗右上角的 × 按钮，弹窗应该会平滑关闭

步骤 5：测试自动弹出

稍微调整阈值，触发新的告警，弹窗应该会自动再次弹出

步骤 6：测试自动关闭

将阈值调高到正常水平，所有告警消失后，弹窗应该会自动关闭

🔧 测试方法二：代码注入强制触发告警

如果当前系统指标较低无法触发告警，可以在浏览器控制台执行以下代码：

// 打开监控页面后，按 F12 打开控制台，执行以下代码：
const store = document.querySelector('#app')._vnode.component.proxy.$pinia.state.value.monitor
store.payload.host.cpu_percent = 95
store.payload.host.mem_percent = 90
store.payload.host.gpu.load = 92
代码已复制到剪贴板！

✅ 阈值保存功能测试

步骤 1：调整阈值

拖动阈值滑块到任意值

步骤 2：点击保存

点击阈值设置右上角的"保存设置"按钮，会提示保存成功

步骤 3：刷新页面

刷新页面后，阈值应该保持为你保存的值

🎯 功能特性验证清单

🐛 如果没有看到弹窗

请检查：
  1. 是否确实有告警产生（阈值低于当前指标）
  2. 浏览器控制台是否有错误信息
  3. 是否被浏览器的弹窗拦截器拦截
  4. 尝试刷新页面后重新测试
